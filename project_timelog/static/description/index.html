Project timelog

Tracking Workflow

Online Demo

The timer allows to immobilize the user's time of working on subtask, helps to follow the volume of work during day and week.

Installation

To install the module fo the following:

  • Install this module in a usual way
Settings
  • Open menu "settings"
  • Open "timelog" in section of Configuration
  • Choose necessary configuration settings and press "apply"

Configuration settings

  • "Set warning time" - is warning time. It must warn a user with audio signal about when timer it stops (in what time it is stops).
  • "Set completion time" - is a limit on timer work for subtask, it stops after achieving this time.
  • "Set normal time" - is warning time for the number of worked hours for current day (a set number of hours).
  • "Set good time" - is warning time for the number of worked hours for current day ( more than a set number of hours).
  • "Set normal time" - is warning time for the number of worked hours for current week (a set number of hours).
  • "Set good time" - is warning time for the number of worked hours for current week ( more than a set number of hours).

Usage

After the installation of the module, the upper right corner will show "00:00:00 00:00 00:00 00:00".

The timer allows to immobilize the user’s time of working on subtask in menu "Project->Task", helps to follow the volume of work during day and week.

The result of timer work in written in menu "Project -> Timelog" in section "My timelog". This section shows logs of current users only and explains which task is for subtask, when the timer was launched and stopped, how long it worked and in what state was task. There is a "time correction" which allows to correct the time (see more in section Correction of time). Manager has the access "timelog" in which logs of all users are seen.

Start timer

While working you need to create a new subtask in page with current task. Press "play" to start the timer. Press "stop" to stop it. After its stopping a general time appears in above the subtask in "Time Spent".

If the subtask was created in one state of task, then timer for this subtask cannot be launched in other state of current task.

If timer was launched in one state of task and state of task has changed while working, then timer automatically stops and creates a new task with the same name.

Checking user’s activity is occurs due to his status in chat. Thus, timer automatically stops, if user is "offline" or there is no internet connection. If new connection appears you need to restart the timer. If timer is stopped user will hear a audio signal.

If there are subtasks with logs of another day, then subtask is unavailable for user. User can start subtasks in which in "Done by" is stated user’s name.

Description of timer work

  • The first timer - is time for current subtask. When approaching the mark of 2 hours (set in Configuration) it changes color to orange and emits a short signal. Upon exceeding mark of 2 hours (set in Configuration) the timer is stopped and the color changes to red, it flashes and emits a long beep. When you click on a timer it works as a pause.
  • The second timer - is general time for current task (only the current user data are summed). Upon exceeding "initially planned hours" (taking into account all the logs for the task) it changes color to yellow. Upon exceeding "initially planned hours" 100% (set) it changes to red. Clicking on the timer opens the current task.
  • The third timer - is general time for current day. Upon reaching 5 hours (set in Configuration) it changes color to yellow. Upon reaching 6 hours (set in Configuration) it changes color to green. Clicking on the timer opens a page with the current day logs.
  • The fourth timer - is general time for current week. Upon reaching 30 hours (set in Configuration) it changes to green and emits a beautiful melody. When reaching 40 hours (set in Configuration) it changes color to blue and emits a long beautiful melody. Clicking on the timer opens a page with logs of current week.

Conditions for timer

  • Simultaneously you may switch on timer for one subtask only.
  • Do not switch on timer for subtasks which contains not today’s logs.
  • Do not create subtasks with the same names in task of one state.

Time correction

If user opened timer occasionally, delete all the unnecessary time with "Time correction". Go to "My timelog" and press log which contains a mistake to change working hours. Press "edit" and "time correction" will be available for editing. User can only reduce time and manager can both reduce it or increase it. If user does not put "-" it will be put automatically.

Compulsory completion of the task

In case of user should immediately goes to a new task in "Stopline", must specify the date and time up to which the user can work. When reaching it he cannot add timelogs to the current task, till Stopline will be changed or deleted. "Stopline" used in difficult tasks and can be edited only by manager. When reaching to Stopline timer changes to orange, and appears the signal.

Prohibition of logs In determined condition of task

To prohibit creating logs in determined condition of task state "task_allow_logs" with the meaning "False".

Uninstallation

Press "uninstall" in module "Project timelog" to delete module.

Note

  • To use the module, you need to be sure that your odoo instance support longpolling, i.e. Instant Messaging works.Read more about how to use the "longpolling"
  • While doing reinstallation of module after its removal clear browser data.
